Full Implementation Spec for Codex
Overview
A HACS-compatible Home Assistant custom integration called google_assistant_manager that provides a UI panel for managing which entities are exposed to Google Home when using the manual (non-Nabu Casa) google_assistant integration. It owns and writes a single generated YAML file that the user includes once in their configuration.yaml.

One-Time User Setup
The user adds this to their configuration.yaml once:
yamlgoogle_assistant:
  project_id: YOUR_PROJECT_ID
  service_account: !include service_account.json
  entity_config: !include google_assistant_entity_config.yaml
```

The integration creates and manages `google_assistant_entity_config.yaml` in the HA config directory from that point forward.
```

yaml_writer.py
Responsible for rendering the store data to google_assistant_entity_config.yaml.
pythonimport yaml
import os

HEADER = "# Auto-generated by google_assistant_manager integration.\n# Do not edit manually — use the Google Assistant panel in Home Assistant.\n\n"

async def write_config(hass, store_data: dict) -> None:
    output_path = os.path.join(hass.config.config_dir, OUTPUT_FILENAME)
    entities = store_data.get("entities", {})
    
    config = {}
    for entity_id, cfg in entities.items():
        entry = {}
        if not cfg.get("expose", True):
            entry["expose"] = False
        if cfg.get("name"):
            entry["name"] = cfg["name"]
        if cfg.get("aliases"):
            entry["aliases"] = cfg["aliases"]
        if entry:  # only write entities that have non-default config
            config[entity_id] = entry
    
    content = HEADER + yaml.dump(config, default_flow_style=False, allow_unicode=True)
    
    def _write():
        with open(output_path, "w", encoding="utf-8") as f:
            f.write(content)
    
    await hass.async_add_executor_job(_write)

config_flow.py
Two steps:
Step 1 — Confirm: Single confirmation form. Text explains the user must add the !include directive to their configuration.yaml. Provides the exact YAML snippet to copy.
Step 2 — Optional import: If google_assistant_entity_config.yaml already exists OR if configuration.yaml contains an entity_config: block inline (not !include), offer to import existing config into the store. Parse it with yaml.safe_load. If import is declined or nothing exists, create empty store.
Only one instance allowed (_async_current_entries check).

websocket_api.py
Register these commands:
google_assistant_manager/get_entities

Fetch all states from hass.states.async_all()

```
### `google_assistant_manager/update_entity`
Schema: `entity_id (str), expose (bool, optional), aliases (list[str], optional), name (str | null, optional)`
- Calls `store.async_update_entity(...)`
- Calls `write_config(...)`
- Calls `hass.services.async_call("homeassistant", "reload_all")` — this reloads all YAML-based integrations including google_assistant. Requires HA 2023.4+.
- Returns `{"success": true}`

### `google_assistant_manager/bulk_update`
Schema: `entity_ids (list[str]), expose (bool)`
- Calls `store.async_bulk_update(...)`
- Calls `write_config(...)`
- Calls reload_all
- Returns `{"success": true, "updated": count}`

### `google_assistant_manager/get_config_snippet`
- Returns the exact YAML the user needs to add to their `configuration.yaml` as a string — useful to display in the UI for first-time users.

## Frontend Panel (`google-assistant-manager-panel.js`)

Single-file vanilla Lit (using the version bundled with HA, imported as `LitElement` from HA's built-in). No external dependencies, no build step required — this keeps it simple and means no bundler/node toolchain is needed for contributors.
```

Behaviour:

On load: calls google_assistant_manager/get_entities WS command
Changes are held locally (dirty state) with an unsaved indicator in the header
Save button batches all dirty changes — calls update_entity per changed entity (or bulk_update where applicable), then calls reload_all once at the end (not per-entity)
Aliases edited as a comma-separated text input, split on save
Domain headers show exposed count and domain-level All/None buttons
Search filters entity_id and friendly_name
First-time banner: if no entities are configured yet, shows the !include snippet from get_config_snippet
require_admin: true on the panel, so non-admin users never see it


Reload Behaviour
Important: homeassistant.reload_all reloads ALL YAML-configured integrations, not just google_assistant. This is the only clean option available without a full restart. Document this clearly in the README. In practice it's fast (a few seconds) and non-disruptive.
The UI should show a toast/spinner after save: "Saving and reloading Google Assistant config…" and confirm when done.

Does not interact with Nabu Casa or the Voice Assistant settings section
Does not require Node.js, a build step, or any Python packages beyond HA's stdlib
Does not store any cloud credentials
